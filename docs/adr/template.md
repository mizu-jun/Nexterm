# ADR-NNNN: タイトル

## ステータス

提案中 / 採用 (YYYY-MM-DD) / 廃止 / 代替済み (ADR-XXXX で置換)

## コンテキスト

この決定が必要になった背景・状況・問題を記述する。
読み手は当時の制約と動機を理解できる必要がある。

例:
- 何の課題に直面したか
- 誰がこの決定に影響を受けるか
- どんな制約があったか（時間・互換性・依存関係 等）

## 決定

何を選んだか、簡潔に。

`「我々は X を採用する」` のような能動形で書くと明快。

## 影響

### ポジティブ
- この決定により得られる利点
- 解決される問題

### ネガティブ
- トレードオフ・追加コスト
- 将来的に再検討が必要になる可能性のあるポイント

## 代替案

検討したが採用しなかった選択肢を簡潔に列挙する。
「なぜ採用しなかったか」を 1 行ずつ。

- **代替案 A**: 採用しなかった理由
- **代替案 B**: 採用しなかった理由
